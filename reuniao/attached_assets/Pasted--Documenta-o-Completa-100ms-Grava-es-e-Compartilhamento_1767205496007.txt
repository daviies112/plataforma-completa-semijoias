## Visão Geral

O 100ms oferece múltiplas opções de gravação e streaming para capturar sessões de videoconferência. As gravações podem ser iniciadas de várias formas:
- Via Dashboard (configuração de template)
- Via Client SDK (iOS, Android, Web, Flutter, React Native)
- Via Server-side APIs REST

### Conceitos Principais
- **Recording Job**: O processo de gravação em execução
- **Recording Asset**: Os arquivos gerados pela gravação (vídeo, chat, transcrições, resumos)
- **Beam**: Bot automatizado que se junta à sala para gravar/transmitir

---

## Tipos de Gravação

### 1. **Browser Composite Recording (Room Composite)**
Grava a perspectiva de um participante visualizando a sala.

**Características:**
- Gera arquivo MP4 único
- Baseado em navegador (bot abre URL da reunião)
- Personalizável via UI customizada
- Resolução configurável (500x480 a 1920x1080)
- Suporta auto-start quando primeiro peer entra

### 2. **SFU Recording (Track Recording)**
Grava streams individuais de cada participante.

**Características:**
- Arquivo MP4 separado por peer
- Útil para composição personalizada offline
- Dois modos: Individual e Composite
- Tamanho estimado: `bitrate (kbps) * 0.45 MB/hora`

### 3. **HLS VOD Recording**
Grava streams HLS para reprodução sob demanda.

**Características:**
- Formato HLS (M3U8 + chunks)
- Adaptive Bitrate (ABR)
- Ideal para casos de uso VOD
- Pode gerar thumbnail automaticamente

### 4. **Server Recording**
Gravação automática de arquivamento.

**Características:**
- Não pode ser parado por SDK
- Configurável apenas via Dashboard
- Propósito de arquivamento

## Gravação Browser Composite

### Configuração via Dashboard

1. Acesse o template no Dashboard
2. Navegue até a aba "Recording"
3. Configure:
   - **Enable auto-start**: Inicia automaticamente quando primeiro peer entra
   - **Resolution**: Largura e altura do vídeo
   - **Maximum Duration**: Duração máxima da gravação
   - **Storage**: 100ms storage ou cloud próprio (S3, GCS, Alibaba OSS)

### Iniciar Gravação via Web SDK

```javascript
// Método básico
try {
  await hmsActions.startRTMPOrRecording({
    record: true,
    meetingURL: 'https://your-app.app.100ms.live/meeting/abc',
    resolution: { width: 1280, height: 720 }
  });
} catch (error) {
  console.error('Erro ao iniciar gravação:', error);
}

// Parar gravação
await hmsActions.stopRTMPAndRecording();
```

## Gravação SFU

### Configuração

1. No Dashboard, acesse o template
2. Na aba "Recording", configure:
   - **Enable SFU Recording**
   - **Upload Info**: Configure bucket S3/GCS/OSS
   - **Recording Mode**: Individual ou Composite

### Estrutura de Arquivos

**Composite Recording:**
```
s3://<bucket>/<prefix>/<room_id>/<date>/<session_id>/Rec-<session_id>-<epoch>.mp4
```

**Individual Recordings:**
```
s3://<bucket>/<prefix>/<room_id>/<date>/<session_id>/<peer_id>/<stream-id>/<stream-id>.mp4
```

### Cálculo de Tamanho

```
Tamanho = (video bitrate + audio bitrate) * 0.45 MB/hora

Exemplo:
- Vídeo 360p a 300kbps = 300 * 0.45 = 135 MB/hora
- Áudio padrão 32kbps = 32 * 0.45 = 15 MB/hora
- Total = 150 MB/hora por peer
```

## HLS Streaming e Gravação

### O que é HLS?

HLS (HTTP Live Streaming) permite escalar para milhões de viewers com latência próxima ao real-time (<10s). O 100ms gera uma URL HLS que pode ser reproduzida em qualquer player compatível.

### Iniciar HLS via Web SDK

```javascript
// HLS básico (default UI)
await hmsActions.startHLSStreaming();

// HLS customizado com gravação
const hlsConfig = {
  meetingURLVariants: [
    {
      meetingURL: 'https://your-app.app.100ms.live/streaming/abc',
      metadata: 'hls-variant-1'
    }
  ],
  recording: {
    singleFilePerLayer: false, // Um MP4 por camada de resolução
    videoOnDemand: true         // Habilita VOD recording (HLS format)
  }
};

await hmsActions.startHLSStreaming(hlsConfig);

// Parar HLS
await hmsActions.stopHLSStreaming();

// Obter URL do stream
const hlsState = hmsStore.getState(selectHLSState);
if (hlsState.running && hlsState.variants.length > 0) {
  const hlsURL = hlsState.variants[0].hlsStreamUrl;
  console.log('HLS URL:', hlsURL);
}
```

            hmsActionResultListener = this
        )

}

// Verificar status
override fun onRoomUpdate(type: HMSRoomUpdate, hmsRoom: HMSRoom) {
    if (type == HMSRoomUpdate.HLS_STREAMING_STATE_UPDATED) {
        val isStreaming = hmsRoom.hlsStreamingState?.running ?: false
        val hlsUrl = hmsRoom.hlsStreamingState?.variants?.firstOrNull()?.hlsStreamUrl
    }
}

### Configurações de Gravação HLS

#### VOD Recording (Video on Demand)
- **Formato**: HLS (M3U8 + chunks de vídeo)
- **Upload Format**: 
  - **Unzipped**: Arquivos enviados diretamente para storage (recomendado)
  - **Zipped**: Todos os arquivos em um ZIP único

#### Single File Per Layer
- **true**: Gera um arquivo MP4 para cada camada de resolução
- **false**: Mantém formato HLS nativo

### Resoluções HLS Suportadas

Configure múltiplas camadas de resolução (144p a 1080p):
```javascript
// Exemplo de configuração no Dashboard

- 480p (854x480) - Baixa qualidade
- 360p (640x360) - Muito baixa
- 240p (426x240)
- 144p (256x144)
```

## RTMP Streaming e Gravação

### O que é RTMP?

RTMP permite transmitir para plataformas como YouTube, Facebook, Twitch, LinkedIn simultaneamente (máximo 3 URLs).

### Iniciar RTMP via Web SDK

```javascript

  meetingURL: 'https://your-app.app.100ms.live/meeting/abc',
  rtmpURLs: [
    'rtmp://a.rtmp.youtube.com/live2/your-stream-key',
    'rtmp://live-api-s.facebook.com/rtmp/your-stream-key'
  ],
  record: true,
  resolution: { width: 1280, height: 720 }
};

// Iniciar streaming e gravação
await hmsActions.startRTMPOrRecording(rtmpConfig);

// Parar
await hmsActions.stopRTMPAndRecording();

// Verificar status
const rtmpState = hmsStore.getState(selectRTMPState);

```

### Android SDK

#### Configuração

1. Adicionar permissão no `AndroidManifest.xml`:
```xml
<uses-permission android:name="android.permission.FOREGROUND_SERVICE" />
```

2. Habilitar screen share no Dashboard para a role

#### Implementação

```kotlin
class ScreenShareActivity : AppCompatActivity() {
    
    private lateinit var hmsSDK: HMSSDK
    
    // Launcher para permissão
    private val resultLauncher = registerForActivityResult(
        ActivityResultContracts.StartActivityForResult()
    ) { result ->
        if (result.resultCode == Activity.RESULT_OK) {
            val mediaProjectionData: Intent? = result.data
            if (mediaProjectionData != null) {
                startScreenShare(mediaProjectionData)
            }
        }
    }
    
    fun requestScreenShare() {
        val mediaProjectionManager = getSystemService(
            Context.MEDIA_PROJECTION_SERVICE
        ) as MediaProjectionManager
        
        resultLauncher.launch(
            mediaProjectionManager.createScreenCaptureIntent()
        )
    }
    
    private fun startScreenShare(intent: Intent) {
        // Notificação customizada (opcional)
        val notification = NotificationCompat.Builder(this, CHANNEL_ID)
            .setContentTitle("Compartilhamento de Tela")
            .setContentText("Sua tela está sendo compartilhada")
            .setSmallIcon(R.drawable.ic_screen_share)
            .build()
        
        hmsSDK.startScreenshare(
            object : HMSActionResultListener {
                override fun onSuccess() {
                    Log.d(TAG, "Screen share iniciado")
                }
                
                override fun onError(error: HMSException) {
                    Log.e(TAG, "Erro: ${error.message}")
                }
            },
            intent,
            notification // Opcional
        )
    }
    
    fun stopScreenShare() {
        hmsSDK.stopScreenshare(object : HMSActionResultListener {
            override fun onSuccess() {
                Log.d(TAG, "Screen share parado")
            }
            
            override fun onError(error: HMSException) {
                Log.e(TAG, "Erro: ${error.message}")
            }
        })
    }
    
    override fun onDestroy() {
        super.onDestroy()
        // IMPORTANTE: Sempre chamar leave para parar o foreground service
        hmsSDK.leave()
    }
}
```

## Transcrição e Resumo

### Tipos de Transcrição

#### 1. Live Transcription (Legendas ao Vivo)

Transcrição em tempo real durante conferências WebRTC.

**Habilitar via Web SDK:**

```javascript
import { HMSTranscriptionMode } from '@100mslive/react-sdk';

// Verificar se transcrição está habilitada
const isTranscriptionEnabled = hmsStore.getState(
  selectIsTranscriptionEnabled
);

// Verificar se tem permissão para iniciar
const canStartTranscription = hmsStore.getState(
  selectIsTranscriptionAllowedByMode(HMSTranscriptionMode.CAPTION)
);

// Iniciar legendas
async function startCaptions() {
  try {
    await hmsActions.startTranscription({
      mode: HMSTranscriptionMode.CAPTION
    });
    console.log('Legendas iniciadas');
  } catch (error) {
    console.error('Erro:', error);
  }
}

// Parar legendas
async function stopCaptions() {
  try {
    await hmsActions.stopTranscription(HMSTranscriptionMode.CAPTION);
    console.log('Legendas paradas');
  } catch (error) {
    console.error('Erro:', error);
  }
}

// Receber transcrições
import { useTranscript } from '@100mslive/react-sdk';

function CaptionComponent() {
  const [captions, setCaptions] = useState([]);
  
  useTranscript({
    onTranscript: (transcripts) => {
      // transcripts é um array de objetos HMSTranscript
      setCaptions(transcripts);
    },
    handleError: (error) => {
      console.error('Erro na transcrição:', error);
    }
  });
  
  return (
    <div>
      {captions.map((caption, index) => (
        <div key={index}>
          <span>{caption.peer_id}: </span>
          <span>{caption.transcript}</span>
          {!caption.final && <span> ...</span>}
        </div>
      ))}
    </div>
  );
}
```

**Objeto HMSTranscript:**

```typescript
interface HMSTranscript {
  start: number;        // Tempo de início em segundos
  end: number;          // Tempo de fim em segundos
  peer_id: string;      // ID do participante falando
  final: boolean;       // true quando transcrição está finalizada
  transcript: string;   // Texto da legenda
}
```

#### 2. HLS Live Transcription

Transcrição ao vivo durante streams HLS.

**Características:**
- Latência: 500ms a 1s
- Formato: WebVTT
- Suporta múltiplos players HLS
- Não inclui identificação de falante

**Habilitar via Dashboard:**

1. Acesse o template
2. Aba "Transcription"
3. Ative "Live Transcription for HLS"
4. Configure:
   - **Custom Vocabulary**: Palavras personalizadas
   - **Language**: Idioma (atualmente apenas inglês)

**Iniciar HLS com Transcrição:**

```javascript
// Web SDK
const hlsConfig = {
  meetingURLVariants: [
    {
      meetingURL: 'https://your-app.app.100ms.live/streaming/abc',
      metadata: 'hls-variant'
    }
  ],
  transcription: {
    enabled: true,
    outputModes: ['txt'], // Não aplicável ao HLS
    customVocabulary: ['palavras', 'personalizadas']
  }
};

await hmsActions.startHLSStreaming(hlsConfig);

// Obter URL da transcrição WebVTT
const hlsState = hmsStore.getState(selectHLSState);
if (hlsState.running && hlsState.variants.length > 0) {
  const variant = hlsState.variants[0];
  const hlsURL = variant.hlsStreamUrl;
  const captionURL = variant.playlistUrl.replace('.m3u8', '_captions.vtt');
}
```

**Reproduzir com hls.js:**

```javascript
import Hls from 'hls.js';

const video = document.getElementById('video');
const hls = new Hls();

hls.loadSource(hlsURL);
hls.attachMedia(video);

// Adicionar track de legenda
const track = document.createElement('track');
track.kind = 'captions';
track.label = 'English';
track.srclang = 'en';
track.src = captionURL;
track.default = true;
video.appendChild(track);
```

#### 3. Post-Call Transcription

Transcrição após a gravação com identificação de falante.

**Características:**
- Suporta identificação de falante (speaker labels)
- Formatos: TXT, SRT, JSON
- Processamento: Mínimo 5min + 20% da duração
- Apenas inglês (mais idiomas em breve)

**Formatos de Saída:**

**Plain Text (.txt):**
```
Speaker 1: Olá, como você está?
Speaker 2: Estou bem, obrigado!
Speaker 1: Ótimo, vamos começar a reunião.
```

**SubRip (.srt):**
```
1
00:00:00,000 --> 00:00:03,500
Speaker 1: Olá, como você está?

2
00:00:03,500 --> 00:00:06,000
Speaker 2: Estou bem, obrigado!

3
00:00:06,000 --> 00:00:09,500
Speaker 1: Ótimo, vamos começar a reunião.
```

**JSON Estruturado (.json):**
```json
{
  "transcription": [
    {
      "speaker": "Speaker 1",
      "start": 0.0,
      "end": 3.5,
      "text": "Olá, como você está?"
    },
    {
      "speaker": "Speaker 2",
      "start": 3.5,

      "text": "Estou bem, obrigado!"
    },
    {
      "speaker": "Speaker 1",

      "end": 9.5,
      "text": "Ótimo, vamos começar a reunião."
    }
  ],
  "duration": 9.5,
  "language": "en"
}
```

          metadata: 'hls-1080p'
        }],

          videoOnDemand: true
        },

          output_modes: ['txt', 'json'],

            sections: [

              { title: 'Ações', format: 'action_items' }
            ]
          }
        }
      };
      
      await hmsActions.startHLSStreaming(config);
      alert('HLS iniciado com gravação!');
    } catch (error) {
      console.error('Erro:', error);
    }
  };
  
  const stopAll = async () => {
    try {
      if (isRecording || isRTMPStreaming) {
        await hmsActions.stopRTMPAndRecording();
      }
      if (isHLSStreaming) {
        await hmsActions.stopHLSStreaming();
      }
      alert('Tudo parado!');
    } catch (error) {
      console.error('Erro:', error);
    }
  };
  
  return (
    <div style={{ padding: '20px' }}>
      <h2>Dashboard de Gravação</h2>
      
      <div style={{ marginBottom: '20px' }}>
        <h3>Status</h3>
        <p>Browser Recording: {isRecording ? '🔴 Gravando' : '⚪ Parado'}</p>
        <p>HLS Streaming: {isHLSStreaming ? '🔴 Ao Vivo' : '⚪ Parado'}</p>
        <p>RTMP Streaming: {isRTMPStreaming ? '🔴 Transmitindo' : '⚪ Parado'}</p>
        
        {isHLSStreaming && hlsState.variants.length > 0 && (
          <div>
            <p>HLS URL: </p>
            <input 
              type="text" 
              value={hlsState.variants[0].hlsStreamUrl} 
              readOnly 
              style={{ width: '100%' }}
            />
          </div>
        )}
      </div>
      
      <div>
        <button onClick={startBrowserRecording} disabled={isRecording}>
          Iniciar Gravação Browser
        </button>
        
        <button onClick={startHLSWithRecording} disabled={isHLSStreaming}>
          Iniciar HLS + Gravação
        </button>
        
        <button onClick={stopAll} disabled={!isRecording && !isHLSStreaming && !isRTMPStreaming}>
          Parar Tudo
        </button>
      </div>
    </div>
  );
}
