# 🔧 Guia de Implementação - Correção do Erro 502 e Vídeo Sumindo

## 📋 Problema Identificado

O erro ocorre porque o `HMSReactiveStore` estava sendo criado como **singleton global** no módulo, causando:

1. ❌ Conflitos entre múltiplas instâncias do componente
2. ❌ AudioContext sendo fechado prematuramente
3. ❌ Vídeo funcionando no lobby mas sumindo na reunião
4. ❌ Erro 502 que derruba o servidor Vite

## ✅ Solução Implementada

### Mudanças Principais

#### 1. **Instância Isolada por Componente**
```typescript
// ❌ ANTES (Global - Problemático)
const hmsManager = new HMSReactiveStore();
const hmsStore = hmsManager.getStore();
const hmsActions = hmsManager.getActions();

// ✅ DEPOIS (Ref - Isolado)
const hmsManagerRef = useRef<HMSReactiveStore | null>(null);
const [hmsStore, setHmsStore] = useState<any>(null);
const [hmsActions, setHmsActions] = useState<any>(null);
```

#### 2. **Limpeza Correta do Lobby**
```typescript
// ✅ Parar TODOS os tracks do preview antes de entrar na reunião
if (streamRef.current) {
  streamRef.current.getTracks().forEach(track => {
    track.stop();
  });
  streamRef.current = null;
}
```

#### 3. **Attach/Detach de Vídeo com Segurança**
```typescript
// ✅ Apenas attach quando track estiver pronto
useEffect(() => {
  if (!videoElement || !videoTrack?.id) return;
  
  const attachVideo = async () => {
    try {
      await hmsActions.attachVideo(videoTrack.id, videoElement);
    } catch (error) {
      console.warn("Error attaching video:", error);
    }
  };
  
  attachVideo();
  
  return () => {
    // Detach seguro
    if (videoTrack?.id) {
      hmsActions.detachVideo(videoTrack.id, videoElement).catch(() => {});
    }
  };
}, [videoTrack]);
```

## 🐛 Troubleshooting

### Problema: Ainda mostra erro de AudioContext

**Solução:**
```bash
# Limpar completamente o cache
rm -rf node_modules/.vite
rm -rf client/dist

# Reinstalar dependências
npm install

# Reiniciar servidor
npm run dev
```

### Problema: Vídeo não aparece no lobby

**Verificar:**
1. Permissões do navegador (câmera/microfone)
2. Console do navegador para erros
3. Dispositivos conectados corretamente

```javascript
// Adicionar debug no console
navigator.mediaDevices.enumerateDevices().then(devices => {
  console.log("Dispositivos:", devices);
});
```

### Problema: Erro 502 ainda ocorre

**Possíveis causas:**
1. Servidor Vite não reiniciou corretamente
2. Cache não foi limpo
3. Outra instância rodando na porta

```bash
# Matar processos na porta 5000
lsof -ti:5000 | xargs kill -9

# Reiniciar
npm run dev
```

### Problema: Vídeo não aparece para outros participantes

**Verificar:**
1. Token 100ms válido
2. Room ID correto
3. Configuração de peer no backend

## 🎯 Diferenças Chave da Solução

| Aspecto | ❌ Antes | ✅ Depois |
|---------|---------|----------|
| HMS Store | Singleton global | Instância por componente via useRef |
| Video Attach | Sem verificação de track | Aguarda track.id antes de attach |
| Cleanup | Incompleto | Para todos os tracks corretamente |
| Error Handling | Causa crash | Try-catch com fallback seguro |
| Lobby → Sala | Stream conflitante | Para stream antes de entrar |

## 📈 Melhorias Adicionais Implementadas

1. **Loading State**: Mostra "Entrando na reunião..." durante conexão
2. **Error Boundary**: Exibe erro amigável sem crashar
3. **Debug Logs**: Console logs para facilitar troubleshooting
4. **Responsive Grid**: Layout adapta ao número de participantes
5. **Mirror Effect**: Vídeo próprio espelhado (padrão UX)

## ⚠️ Avisos Importantes

### 1. React 19 Compatibility
Se atualizar para React 19, monitore:
- Mudanças no timing de useEffect cleanup
- Strict Mode comportamento (double render)

### 2. Replit Plugin
O plugin `@replit/vite-plugin-runtime-error-modal` pode:
- Amplificar erros de runtime
- Considere remover em produção

### 3. Produção vs Desenvolvimento
Esta correção funciona em ambos ambientes, mas teste em produção com:
```bash
npm run build
npm run preview
```
