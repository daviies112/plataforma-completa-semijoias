## Visão Geral

O 100ms oferece diferentes modos de gravação para capturar sessões de videoconferência:

### Modos de Gravação Disponíveis

**1. Composite Recordings**
- Grava uma única asset composta combinando tracks de mídia de todos os peers
- Tipos: Browser Composite e Custom Composite
- Output: Arquivo MP4 único

**2. Track Recordings**  
- Grava cada track de áudio e vídeo de cada peer como assets separadas
- Útil para edição pós-produção

**3. Stream Recordings**
- Grava os streams de mídia de cada peer (áudio e vídeo combinados) como assets separadas

**4. HLS Live Stream Recording**
- Captura feed contínuo de áudio/vídeo durante evento ao vivo
- Foco em baixa latência
- Formatos: VOD (Video-On-Demand) ou MP4 por camada

### Conceitos Chave

**Beam**: Bot automatizado que entra na sala como peer para gravar/transmitir. É um navegador automatizado que abre a URL da reunião.

**Recording Job**: O processo de gravação em execução

**Recording Asset**: Os arquivos gerados após o processamento (vídeo, chat, transcrições, resumos)

**Template**: Coleção de roles, configurações de sala e configurações de gravação

## Tipos de Gravação

### Comparação dos Modos de Gravação

**Browser Composite Recording:**
- ✅ Arquivo MP4 único pronto para uso
- ✅ Customizável via URL da web app
- ✅ Auto-start quando primeiro peer entra
- ❌ Requer URL pública acessível

**SFU Recording (Track/Stream):**
- ✅ Arquivos separados por peer
- ✅ Flexibilidade para edição pós-produção
- ✅ Modo Individual ou Composite
- ❌ Requer processamento adicional

**HLS Recording:**
- ✅ Adaptive bitrate (ABR)
- ✅ Ideal para VOD
- ✅ Múltiplas resoluções
- ❌ Formato HLS (requer player compatível)

## Browser Composite Recording

### O que é?

Browser Composite Recording captura a perspectiva de um participante vendo a sala. O 100ms simula um peer join: um navegador automatizado (chamado "Beam") entra na sala como bot peer e grava o que vê e ouve.

### Características

- **Output**: Arquivo MP4 único
- **Resolução padrão**: 1280x720 (16:9)
- **Resolução mínima**: 500x480 (landscape) ou 500x500 (portrait)
- **Resolução máxima**: 1920x1080 (landscape) ou 1080x1920 (portrait)
- **Customizável**: Pode usar qualquer web app via `meeting_url`
- **Audio-only**: Suportado via API

### Formas de Iniciar

#### 1. Auto-Start (Dashboard)

Configure no Dashboard → Template → Recording tab:

```
✅ Enable auto-start on room join
```

**Comportamento:**
- Inicia automaticamente quando primeiro peer entra
- Para automaticamente 5 minutos após todos os peers saírem
- **Nota**: Desabilita HLS e RTMP streaming para esse template

#### 3. Via Server-side API

```bash
curl --request POST \
  'https://api.100ms.live/v2/recordings/room/<room_id>/start' \
  --header 'Content-Type: application/json' \
  --header 'Authorization: Bearer <management_token>' \
  --data-raw '{
    "meeting_url": "https://myapp.app.100ms.live/meeting/room-id?skip_preview=true",
    "resolution": {
      "width": 1280,
      "height": 720
    }
  }'
```

**Regiões da API:**
- **Índia (padrão)**: `https://api.100ms.live/v2/recordings`
- **EUA**: `https://prod-us2.100ms.live/api/v2/recordings`
- **Europa**: `https://prod-eu2.100ms.live/api/v2/recordings`

### Customização da Meeting URL

A `meeting_url` deve levar diretamente para a sala sem requerer ação do usuário.

**Parâmetros importantes:**

```
https://myapp.app.100ms.live/preview/<room_id>/<role>?param1&param2...
```

**Parâmetros disponíveis:**

- `skip_preview=true`: Pula página de preview e entra direto (sem header/controles)
- `skip_preview_headful=true`: Entra direto mas mantém header/controles
- `auth_token=<token>`: Token de autenticação para o beam bot
- `ui_mode=activespeaker`: Abre em modo active speaker (falante atual no centro)
- `name=<name>`: Nome do peer gravador

**Exemplo completo:**

```
https://myapp.app.100ms.live/preview/61101349d0f56e684b608c08/__internal_recorder?skip_preview=true&auth_token=eyJhbGc...&ui_mode=activespeaker
```

### Customização de Role (Recorder Role)

Crie uma role específica para o bot de gravação no Dashboard:

**Configuração da role "recorder":**

1. Acesse Template → Roles → Add Role
2. Nome: `recorder`
3. **Publish Strategies**: NENHUMA (bot não publica áudio/vídeo)
4. **Subscribe Strategies**: Válidas (bot subscreve às outras roles)

**Benefícios:**
- Controlar o que o bot vê
- UI customizada para gravação
- Bot não aparece na contagem de participantes normais

### Gravação Audio-Only

Para gravar apenas áudio (sem vídeo):

**Via API:**

```bash
curl --request POST \
  'https://api.100ms.live/v2/recordings/room/<room_id>/start' \
  --header 'Authorization: Bearer <management_token>' \
  --data-raw '{
    "meeting_url": "https://myapp.app.100ms.live/meeting/room-id",
    "audio_only": true
  }'
```

### Resolução Customizada

**Landscape:**
- Padrão: 1280x720 (16:9)
- Mínimo: 500x480
- Máximo: 1920x1080

**Portrait (Mobile-first):**
- Exemplo: 720x1280 (9:16)
- Mínimo: 500x500
- Máximo: 1080x1920

### Best Practices

**✅ DO:**
- Use `skip_preview=true` na meeting URL
- Configure role sem permissões de publish para o recorder
- Habilite logs do SDK na web app
- Use auth_token para autenticar o beam bot
- Configure storage personalizado (S3/GCS/OSS)

**❌ DON'T:**
- Não use URLs que requerem clique em botão "Join"
- Não desabilite logs do SDK (beam precisa para debug)
- Não use resolução maior que necessário (aumenta tamanho do arquivo)
- Não confie no storage do 100ms para produção (apenas 15 dias)

## SFU Recording

### O que é?

SFU Recording grava tracks individuais de cada peer, gerando arquivos separados. Ideal para edição pós-produção.

### Tipos

**1. Individual Recording:**
- Um arquivo MP4 por track (áudio e vídeo separados)
- Estrutura: `<peer_id>/<stream-id>/<stream-id>.mp4`

**2. Composite Recording:**
- Um arquivo MP4 por peer (áudio e vídeo combinados)
- Estrutura: `<peer_id>/composite.mp4`

### Configuração no Dashboard

1. Acesse Template → Recording tab
2. Enable "SFU Recording"
3. Configure Upload Info (S3/GCS/OSS)
4. Escolha Recording Mode: Individual ou Composite

### Estrutura de Arquivos no Storage

**Path base:**
```
s3://<bucket>/<prefix>/<room_id>/<date>/<session_id>/<peer_id>/<stream-id>/<stream-id>.mp4
```

**SFU Recording (Composite):**
```
s3://<bucket>/<prefix>/<room_id>/<date>/<session_id>/<peer_id>/composite.mp4
```

**HLS Recording:**
```
s3://<bucket>/<prefix>/hls/<room_id>/<session_id>/master.m3u8
s3://<bucket>/<prefix>/hls/<room_id>/<session_id>/1080p/index.m3u8
s3://<bucket>/<prefix>/hls/<room_id>/<session_id>/1080p/segment_0.ts
```

## Troubleshooting

### Problemas com Browser Recording

#### Gravação não inicia

**Sintoma:** `startRTMPOrRecording()` retorna erro ou gravação fica em "starting"

**Possíveis causas:**

1. **URL não é acessível publicamente**
   - ✅ Solução: Use URL pública acessível pela internet
   - Em ambientes Replit/desenvolvimento, configure proxy reverso adequado

2. **Role não tem permissão de recording**
   - ✅ Solução: No Dashboard → Template → Roles, verifique se role do token tem "Recording" habilitado

3. **Meeting URL requer interação do usuário**
   - ✅ Solução: Use `?skip_preview=true` na URL
   - Certifique-se que não há popup/modal bloqueando join

4. **Auth token inválido ou expirado**
   - ✅ Solução: Gere novo token com role válida
   - Use role específica para recorder (`__internal_recorder`)

**Como verificar:**

```javascript
// Web SDK - Verificar erro
const room = hmsStore.getState(selectHMSRoom);
const error = room.browserRecordingState?.error;

if (error) {
  console.error('Erro de gravação:', error.code, error.message);
}
```

#### Bot não aparece na sala

**Sintoma:** Gravação iniciada mas bot beam não está visível

**Causa:** Bot está presente mas com role de apenas subscribe (sem publish)

**Verificação:**
```javascript
// Verificar se há peer com role "recorder"
const peers = hmsStore.getState(selectPeers);
const recorderPeer = peers.find(p => p.roleName === '__internal_recorder');

if (recorderPeer) {
  console.log('Bot presente:', recorderPeer.id);
}
```

#### Arquivo de gravação vazio ou corrompido

**Possíveis causas:**

1. **Gravação parada muito rápido** (< 5 segundos)
   - ✅ Solução: Aguarde pelo menos 10 segundos antes de parar

2. **Nenhum peer publicando mídia**
   - ✅ Solução: Certifique-se que há pelo menos 1 peer com áudio/vídeo

3. **Erro de upload para storage**
   - ✅ Solução: Verifique credenciais e permissões do bucket
   - Verifique webhook `beam.failure` para detalhes

### Problemas com Screen Share

#### Permissão negada pelo navegador

**Sintoma:** Erro "Permission denied" ou "NotAllowedError"

**Causas:**

1. **Usuário clicou em "Cancel"**
   - Normal, não é erro do código

2. **HTTPS não configurado**
   - ✅ Solução: Screen share requer HTTPS (exceto localhost)

3. **Permissões do navegador bloqueadas**
   - ✅ Solução: Limpar permissões em chrome://settings/content

**Tratamento:**

```javascript
try {
  await hmsActions.setScreenShareEnabled(true);
} catch (error) {
  if (error.message.includes('Permission denied')) {
    alert('Você precisa conceder permissão para compartilhar a tela');
  }
}
```

#### Screen share não aparece para outros peers

**Sintoma:** Local peer vê screen share mas remote peers não

**Possíveis causas:**

1. **Track auxiliar não está sendo renderizado**

```javascript
// Verificar se track existe
const peers = hmsStore.getState(selectPeers);
const sharingPeer = peers.find(p => 
  p.auxiliaryTracks.some(t => t.source === 'screen')
);

if (sharingPeer) {
  const screenTrack = sharingPeer.auxiliaryTracks.find(
    t => t.source === 'screen'
  );
  console.log('Screen track existe:', screenTrack.id);
  
  // Attach ao vídeo element
  const videoEl = document.getElementById('screen-share-video');
  await hmsActions.attachVideo(screenTrack.id, videoEl);
}
```

2. **Role não tem permissão de subscribe para screen share**
   - ✅ Solução: No Dashboard, verifique Subscribe Strategies da role

#### Screen share trava ou tem baixo FPS

**Causas:**

1. **Resolução muito alta**
   - ✅ Solução: Navegador captura em resolução nativa, reduzir qualidade do lado do receiver

2. **CPU sobrecarregada**
   - ✅ Solução: Fechar aplicações desnecessárias

3. **Bandwidth insuficiente**
   - ✅ Solução: Verificar conexão de internet

#### iOS screen share não funciona

**Possíveis causas:**

1. **Broadcast Extension não configurada**
   - ✅ Solução: Seguir setup completo do iOS (App Group, Extension, etc)

2. **App Group ID diferente**
   - ✅ Solução: Usar exatamente o mesmo ID no app e extensão

3. **Extensão não está no target correto**
   - ✅ Solução: Verificar se BroadcastExtension está em Targets do Xcode

### Problemas com HLS

#### HLS não inicia

**Sintoma:** `startHLSStreaming()` falha

**Causas:**

1. **Template não está configurado para HLS**
   - ✅ Solução: Dashboard → Template → Destinations → HLS, habilitar

2. **Meeting URL incorreta**
   - ✅ Solução: Usar URL do tipo `/streaming/` não `/meeting/`

#### URL HLS retorna 404

**Sintoma:** `hlsStreamUrl` retorna 404 quando acessada

**Causa:** HLS leva alguns segundos para iniciar (~10-15s)

**Solução:**
```javascript
// Aguardar HLS estar realmente rodando
const hlsState = hmsStore.getState(selectHLSState);

if (hlsState.running && hlsState.variants[0].startedAt) {
  // Aguardar alguns segundos extras
  setTimeout(() => {
    const url = hlsState.variants[0].hlsStreamUrl;
    console.log('HLS URL:', url);
  }, 5000);
}
```

```jsx
// Componente de screen share
function ScreenShareView() {

  const screenTrack = useHMSStore(

  );

// Componente principal
function ConferenceRoom() {
  const hmsActions = useHMSActions();
  const isConnected = useHMSStore(selectIsConnectedToRoom);
  const peers = useHMSStore(selectPeers);
  const localPeer = useHMSStore(selectLocalPeer);
  const isScreenShared = useHMSStore(selectIsLocalScreenShared);
  const isSomeoneScreenSharing = useHMSStore(selectIsSomeoneScreenSharing);
  const room = useHMSStore(selectHMSRoom);
  
  const [authToken, setAuthToken] = useState('');
  const [userName, setUserName] = useState('');
  
  // Status
  const isRecording = room?.browserRecordingState?.running || false;
  const isHLSStreaming = room?.hlsStreamingState?.running || false;
  const audioEnabled = localPeer?.audioTrack !== undefined;
  const videoEnabled = localPeer?.videoTrack !== undefined;
  
  // Join room
  const joinRoom = async () => {
    if (!authToken || !userName) {
      alert('Preencha token e nome');
      return;
    }
    
    try {
      await hmsActions.join({
        authToken,
        userName,
        settings: {
          isAudioMuted: false,
          isVideoMuted: false
        }
      });
    } catch (error) {
      console.error('Erro ao entrar:', error);
      alert('Erro ao entrar na sala');
    }
  };
  
  // Leave room
  const leaveRoom = () => {
    hmsActions.leave();
  };
  
  // Toggle audio
  const toggleAudio = async () => {
    await hmsActions.setLocalAudioEnabled(!audioEnabled);
  };
  
  // Toggle video
  const toggleVideo = async () => {
    await hmsActions.setLocalVideoEnabled(!videoEnabled);
  };
  
  // Start recording
  const startRecording = async () => {
    try {
      await hmsActions.startRTMPOrRecording({
        record: true,
        meetingURL: window.location.href,
        resolution: { width: 1280, height: 720 }
      });
      console.log('Gravação iniciada');
    } catch (error) {
      console.error('Erro ao gravar:', error);
      alert('Erro ao iniciar gravação');
    }
  };
  
  // Stop recording
  const stopRecording = async () => {
    try {
      await hmsActions.stopRTMPAndRecording();
      console.log('Gravação parada');
    } catch (error) {
      console.error('Erro ao parar:', error);
    }
  };
  
  // Toggle screen share
  const toggleScreenShare = async () => {
    try {
      await hmsActions.setScreenShareEnabled(!isScreenShared);
    } catch (error) {
      console.error('Erro screen share:', error);
      if (error.message.includes('Permission denied')) {
        alert('Você precisa conceder permissão');
      }
    }
  };
  
  // UI de entrada
  if (!isConnected) {
    return (
      <div className="join-screen">
        <h1>Entrar na Reunião</h1>
        <input
          type="text"
          placeholder="Seu nome"
          value={userName}
          onChange={(e) => setUserName(e.target.value)}
        />
        <input
          type="text"
          placeholder="Auth Token"
          value={authToken}
          onChange={(e) => setAuthToken(e.target.value)}

          <div className="recording-indicator">
```

### Estimativa de Tamanho

Fórmula: `(video bitrate + audio bitrate) * 0.45 MB/hora`

- Áudio @ 32kbps: 32 * 0.45 = 15 MB/hora

**Cálculo para sessão:**

## HLS Recording

### O que é?

HLS (HTTP Live Streaming) permite transmitir para milhões de viewers com baixa latência (<10s). O 100ms pode gravar esses streams para VOD.

### Formatos de Gravação HLS

**1. Video-On-Demand (VOD):**
- Formato: HLS nativo (M3U8 + chunks)
- Adaptive bitrate (ABR)
- Ideal para replay
- Pode ser unzipped ou zipped no storage

**2. Per Layer MP4:**
- Arquivo MP4 separado por resolução
- Acesso a footage bruto por qualidade
- Ideal para remixing

### Iniciar HLS com Gravação

**Web SDK:**

```javascript
// HLS básico
await hmsActions.startHLSStreaming();

// HLS com gravação VOD
const config = {
  meetingURLVariants: [{
    meetingURL: 'https://myapp.app.100ms.live/streaming/room-id',
    metadata: 'hls-1080p'
  }],
  recording: {
    singleFilePerLayer: false,  // true = MP4 por layer, false = HLS nativo
    videoOnDemand: true          // Habilita gravação VOD
  }
};

await hmsActions.startHLSStreaming(config);

// Parar HLS
await hmsActions.stopHLSStreaming();

// Obter URL do stream
const hlsState = hmsStore.getState(selectHLSState);
if (hlsState.running && hlsState.variants.length > 0) {
  const hlsURL = hlsState.variants[0].hlsStreamUrl;
  console.log('HLS Stream URL:', hlsURL);
}
```

## RTMP Streaming com Gravação

### O que é?

RTMP permite transmitir simultaneamente para plataformas como YouTube, Facebook, Twitch (máximo 3 URLs).

### Iniciar RTMP + Gravação

**Web SDK:**

```javascript
const config = {
  meetingURL: 'https://myapp.app.100ms.live/meeting/room-id?skip_preview=true',
  rtmpURLs: [
    'rtmp://a.rtmp.youtube.com/live2/<stream-key>',
    'rtmp://live-api-s.facebook.com:80/rtmp/<stream-key>'
  ],
  record: true,  // Habilita gravação junto com streaming
  resolution: { width: 1280, height: 720 }
};

await hmsActions.startRTMPOrRecording(config);

// Parar (para RTMP E gravação)
await hmsActions.stopRTMPAndRecording();

// Verificar status
const room = hmsStore.getState(selectHMSRoom);
const isStreaming = room.rtmpHMSRtmpStreamingState?.running || false;
const isRecording = room.browserRecordingState?.running || false;
```

**Interfaces RTMP:**

```typescript
export enum HMSStreamingState {
  NONE = "none",
  STARTED = "started",
  STOPPED = "stopped",
  FAILED = "failed"
}

export interface HMSRTMP {
  running: boolean;
  startedAt?: Date;
  state?: HMSStreamingState;
  error?: HMSException;
}
```

### URLs RTMP Comuns

**YouTube Live:**
```
rtmp://a.rtmp.youtube.com/live2/<stream-key>
```

**Facebook Live:**
```
rtmp://live-api-s.facebook.com:80/rtmp/<stream-key>
```

**Twitch:**
```
rtmp://live.twitch.tv/app/<stream-key>
```

### Regras Importantes

1. **Não pode iniciar RTMP e gravação separadamente**: Devem ser iniciados juntos
2. **Não pode parar apenas um**: `stopRTMPAndRecording()` para ambos
3. **Máximo 3 RTMP URLs** por vez
4. **Resolução compartilhada**: RTMP e gravação usam a mesma resolução

**Exemplo de uso:**

```javascript
// ✅ CORRETO: Inicia ambos juntos
await hmsActions.startRTMPOrRecording({
  meetingURL: 'https://...',
  rtmpURLs: ['rtmp://...'],
  record: true
});

// ✅ CORRETO: Apenas streaming
await hmsActions.startRTMPOrRecording({
  meetingURL: 'https://...',
  rtmpURLs: ['rtmp://...'],
  record: false  // ou omitir
});

// ✅ CORRETO: Apenas gravação
await hmsActions.startRTMPOrRecording({
  meetingURL: 'https://...',
  record: true
  // rtmpURLs omitido ou vazio
});

// ❌ ERRADO: Não pode adicionar streaming depois
await hmsActions.startRTMPOrRecording({ record: true });
// ... depois ...
await hmsActions.startRTMPOrRecording({ rtmpURLs: ['rtmp://...'] }); // ERRO!
```

**Implementação:**

```kotlin
import android.content.Intent
import android.media.projection.MediaProjectionManager
import androidx.activity.result.contract.ActivityResultContracts

class ScreenShareActivity : AppCompatActivity() {
    
    private lateinit var hmsSDK: HMSSDK
    
    // Launcher para solicitar permissão
    private val screenCaptureResultLauncher = registerForActivityResult(
        ActivityResultContracts.StartActivityForResult()
    ) { result ->
        if (result.resultCode == Activity.RESULT_OK) {
            val data: Intent? = result.data
            if (data != null) {
                startScreenShare(data)
            }
        } else {
            Log.e(TAG, "Usuário negou permissão de screen share")
        }
    }
    
    // Solicitar permissão
    fun requestScreenSharePermission() {
        val mediaProjectionManager = getSystemService(
            Context.MEDIA_PROJECTION_SERVICE
        ) as MediaProjectionManager
        
        val intent = mediaProjectionManager.createScreenCaptureIntent()
        screenCaptureResultLauncher.launch(intent)
    }
    
    // Iniciar screen share
    private fun startScreenShare(mediaProjectionIntent: Intent) {
        // Notificação customizada (opcional, mas recomendado)
        val notification = NotificationCompat.Builder(this, CHANNEL_ID)
            .setContentTitle("Screen Share Ativo")
            .setContentText("Sua tela está sendo compartilhada")
            .setSmallIcon(R.drawable.ic_screen_share)
            .build()
        
        hmsSDK.startScreenshare(
            actionResultListener = object : HMSActionResultListener {
                override fun onSuccess() {
                    Log.d(TAG, "Screen share iniciado com sucesso")
                }
                
                override fun onError(error: HMSException) {
                    Log.e(TAG, "Erro ao iniciar screen share: ${error.message}")
                }
            },
            intent = mediaProjectionIntent,
            notification = notification  // Opcional
        )
    }
    
    // Parar screen share
    fun stopScreenShare() {
        hmsSDK.stopScreenshare(object : HMSActionResultListener {
            override fun onSuccess() {
                Log.d(TAG, "Screen share parado")
            }
            
            override fun onError(error: HMSException) {
                Log.e(TAG, "Erro ao parar: ${error.message}")
            }
        })
    }
    
    // Verificar se está compartilhando
    fun isScreenSharing(peer: HMSPeer): Boolean {
        return peer.auxiliaryTracks.any { it.source == "screen" }
    }
    
    override fun onDestroy() {
        super.onDestroy()
        // IMPORTANTE: Sempre chamar leave para parar o foreground service
        hmsSDK.leave()
    }
}
```

```javascript
const jwt = require('jsonwebtoken');
const { v4: uuidv4 } = require('uuid');

function generateManagementToken() {
  const payload = {
    access_key: 'your-app-access-key',
    type: 'management',
    version: 2,
    iat: Math.floor(Date.now() / 1000),
    nbf: Math.floor(Date.now() / 1000)
  };
  
  const token = jwt.sign(payload, 'your-app-secret', {
    algorithm: 'HS256',
    expiresIn: '24h',
    jwtid: uuidv4()
  });
  
  return token;
}
```

## Recording Assets

### O que são?

Recording Assets são os arquivos gerados após o processamento das gravações.

**Tipos de Assets:**
- `room-composite`: Vídeo MP4 da gravação composite
- `room-vod`: Vídeo HLS (M3U8 + chunks)
- `chat`: Mensagens do chat em JSON
- `transcript`: Transcrição (txt/srt/json)
- `summary`: Resumo gerado por IA (json)
