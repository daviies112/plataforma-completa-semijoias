# 🗄️ BANCO DE DADOS (Supabase)

## Tabela: tenants
```sql
CREATE TABLE tenants (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  nome TEXT NOT NULL,
  slug TEXT UNIQUE NOT NULL,
  email TEXT,
  telefone TEXT,
  logo_url TEXT,
  configuracoes JSONB DEFAULT '{
    "horario_comercial": {
      "inicio": "08:00",
      "fim": "18:00"
    },
    "duracao_padrao": 60,
    "cores": {
      "primaria": "#3B82F6",
      "secundaria": "#10B981"
    }
  }'::jsonb,
  token_100ms TEXT,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

## Tabela: usuarios_tenant
```sql
CREATE TABLE usuarios_tenant (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id UUID REFERENCES tenants(id) ON DELETE CASCADE,
  nome TEXT NOT NULL,
  email TEXT NOT NULL UNIQUE,
  telefone TEXT,
  role TEXT DEFAULT 'user',
  avatar_url TEXT,
  created_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE(tenant_id, email)
);
```

## Tabela: reunioes
```sql
CREATE TABLE reunioes (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id UUID REFERENCES tenants(id) ON DELETE CASCADE,
  usuario_id UUID REFERENCES usuarios_tenant(id),
  nome TEXT NOT NULL,
  email TEXT NOT NULL,
  telefone TEXT,
  titulo TEXT,
  descricao TEXT,
  data_inicio TIMESTAMPTZ NOT NULL,
  data_fim TIMESTAMPTZ NOT NULL,
  duracao INTEGER,
  room_id_100ms TEXT UNIQUE,
  room_code_100ms TEXT,
  link_reuniao TEXT,
  status TEXT DEFAULT 'agendada',
  participantes JSONB DEFAULT '[]'::jsonb,
  gravacao_url TEXT,
  transcricao_id UUID,
  metadata JSONB DEFAULT '{}'::jsonb,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

## Tabela: transcricoes
```sql
CREATE TABLE transcricoes (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  reuniao_id UUID REFERENCES reunioes(id) ON DELETE CASCADE,
  tenant_id UUID REFERENCES tenants(id) ON DELETE CASCADE,
  transcricao_completa TEXT,
  resumo TEXT,
  topicos JSONB,
  acoes JSONB,
  created_at TIMESTAMPTZ DEFAULT now()
);
```

```javascript

  - Envia webhook para n8n

PATCH /api/reunioes/:id - Atualizar reunião
  Body: { data_inicio, data_fim, status }
  - Atualiza sala no 100ms se necessário
  
DELETE /api/reunioes/:id - Cancelar reunião
  - Cancela sala no 100ms
  - Atualiza status para 'cancelada'
  - Envia webhook para n8n

POST /api/reunioes/verificar-disponibilidade
  Body: { data_inicio, data_fim }
  Response: { disponivel: boolean, conflitos: [] }

GET /api/reunioes/:id/token-100ms - Gerar token para participante entrar na reunião
```

## 3. Integração Supabase (server/services/supabase.js)
```javascript
const { createClient } = require('@supabase/supabase-js');

const supabase = createClient(
  process.env.SUPABASE_URL,
  process.env.SUPABASE_SERVICE_KEY
);

class SupabaseService {
  async buscarReuniao(id, tenant_id) {
    const { data, error } = await supabase
      .from('reunioes')
      .select('*')
      .eq('id', id)
      .eq('tenant_id', tenant_id)
      .single();
    
    if (error) throw error;
    return data;
  }
  
  async verificarDisponibilidade(tenant_id, data_inicio, data_fim, excluir_id = null) {
    let query = supabase
      .from('reunioes')
      .select('*')
      .eq('tenant_id', tenant_id)
      .eq('status', 'agendada')
      .or(`data_inicio.lte.${data_fim},data_fim.gte.${data_inicio}`);
    
    if (excluir_id) {
      query = query.neq('id', excluir_id);
    }
    
    const { data, error } = await query;
    
    return {
      disponivel: !data || data.length === 0,
      conflitos: data || []
    };
  }
}
```

# 🎯 FUNCIONALIDADES ESPECÍFICAS

## 1. Sistema Multitenant
- Cada requisição deve incluir tenant_id (extraído do token JWT)
- Middleware para validar acesso ao tenant
- Isolamento total de dados entre tenants

## 2. Fluxo de Agendamento
```
Usuário preenche formulário
  ↓
Frontend valida dados
  ↓
POST /api/reunioes
  ↓
Backend verifica disponibilidade
  ↓
Cria sala no 100ms
  ↓
Salva no Supabase
  ↓
Envia webhook para n8n (confirmação WhatsApp)
  ↓
Retorna dados para frontend
```

## 3. Fluxo de Reunião
```
Usuário clica "Entrar na Reunião"
  ↓
GET /api/reunioes/:id/token-100ms
  ↓
Frontend carrega componente Meeting100ms
  ↓
Join room com token
  ↓
Reunião acontece
  ↓
Ao finalizar: POST /api/webhooks/reuniao-finalizada
  ↓
n8n recebe webhook para transcrição
```
