# Nexus Production Maintenance & Updates 🚀🔧

**Objective:** Zero downtime (where possible) and guaranteed environment stability.

## 1. Hotfix & Update Workflow 🔄
1.  **Stop/Backup (If critical)**: Ensure the current `dist` is stable.
2.  **Pull/Apply Changes**: Apply code changes.
3.  **Build Phase**:
    ```powershell
    npm run build
    ```
    *   This compiles both frontend and backend (`dist/index.mjs`).
4.  **Process Management**:
    *   Identify the current process on port 5000:
        ```powershell
        netstat -ano | findstr :5000
        ```
    *   Restart with production flags:
        ```powershell
        taskkill /F /PID <PID>
        $env:NODE_ENV="production"; $env:PORT="5000"; node dist/index.mjs
        ```

## 2. Health Check Checklist 📋
*   **Port 5000**: Must be LISTENING.
*   **Public API**: Test a known public endpoint (e.g., `/api/public/workspace/test_insert_2`).
*   **Icons/CORS**: Verify that static assets like PWA icons load with HTTP 200.

## 3. Deployment Safety 🛡️
*   **Verify DB Migration**: If database changes were made, run `npm run db:push` before restarting the server.
*   **Cache Clear**: Invalidate server-side caches if configuration changes were applied (e.g., `publicCache.ts`).

## 4. GitHub Sync 📦
*   **Post-Update**: Notify the user when the local/VPS environment is stable so they can push the changes to GitHub.
