1. **Documentos** sÃ£o fragmentados (chunking) em pedaÃ§os com overlap

3. Na hora da query, a pergunta tambÃ©m vira embedding
4. O sistema busca os pedaÃ§os **mais similares semanticamente** (nÃ£o por keyword)
5. Os pedaÃ§os recuperados sÃ£o passados como **contexto** para o LLM responder

### Multi-Query Retrieval
```python

# SOLUÃ‡ÃƒO: gerar mÃºltiplas reformulaÃ§Ãµes e combinar resultados

async def multi_query_retrieval(question: str, vectorstore) -> list:
    # 1. Gerar 3 reformulaÃ§Ãµes da mesma pergunta
    reformulations_prompt = f"""VocÃª Ã© um assistente que gera 3 versÃµes diferentes 
    da seguinte pergunta do usuÃ¡rio para busca em banco de documentos.
    
    Pergunta original: {question}

    for query in queries:

            unique_docs.append(doc)
    
    return unique_docs[:10]  # top 10 Ãºnicos
```

## 4. Reranking: Melhorar a Qualidade do Retrieval ðŸŽ¯

```python
# PROBLEMA: embedding similarity â‰  relevÃ¢ncia real para a pergunta
# SOLUÃ‡ÃƒO: usar um modelo de reranking especializado apÃ³s a busca inicial

from sentence_transformers import CrossEncoder

# Cohere Reranker (cloud) ou Cross-Encoder local
reranker = CrossEncoder('cross-encoder/ms-marco-MiniLM-L-6-v2')

def rerank_documents(query: str, documents: list, top_k: int = 5) -> list:
    # Calcular score de cada par (query, documento)
    pairs = [(query, doc.page_content) for doc in documents]
    scores = reranker.predict(pairs)
    
    # Reordenar por score
    ranked = sorted(zip(scores, documents), reverse=True)
    
    return [doc for score, doc in ranked[:top_k]]

# PIPELINE COMPLETO com Reranking:
# 1. Hybrid search retorna 20 candidatos
# 2. Reranker seleciona os 5 mais relevantes
# Resultado: qualidade muito superior ao embedding similarity sozinho
```
