## 1. The Architecture ðŸ—ï¸
*   **The Hub:** We use **Selenoid** (Go-based Selenium Hub) instead of the heavy Java Grid.
*   **The Nodes:** Every browser session is a fresh **Docker Container**.
    *   *Benefits:* No cache pollution, no "zombie" chrome processes, complete isolation.
    *   *Impact:* We can run 50 parallel tests on a 16GB RAM VPS without crashing.

## 2. The Separation of Powers âš”ï¸
*   **Selenoid (The Web Layer):**
    *   Used for: Clicking buttons, filling forms, extracting text, navigating URLs.
    *   *Why:* It speaks "WebDriver" protocol directly to the browser internal API. 100% reliable.
*   **PyAutoGUI (The OS Layer):**
    *   Used *ONLY* for: Clicking "Save As" windows, handling file upload dialogs, or interacting with non-web apps.
    *   *Rule:* If it has a DOM ID, use Selenoid. If it's a Windows Native Prompt, use PyAutoGUI.

## 3. The "Ghost" Protocol (Headless & Recorded) ðŸ“¹
*   **Default Mode:** Browsers run headless (invisible) to save GPU resources.
*   **Surveillance:** `enableVideo: true` is MANDATORY.
    *   If a bot fails, we don't guess. We watch the `.mp4` video file generated by Selenoid.

## 4. Scaling Standard ðŸ“ˆ
*   **Concurrency:** Limit parallel sessions to `CPU Cores * 2`.
*   **Queueing:** If the fleet is full, requests are queued by Selenoid automatically.
